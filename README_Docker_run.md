# 🚀 Автоматизовані тести (Pytest + Behave) у Docker

Цей проект містить налаштовану інфраструктуру для паралельного запуску UI та API автотестів за допомогою фреймворків **Pytest** та **Behave**. Усі тести виконуються в ізольованих Docker-контейнерах з використанням віддаленого браузера Chrome (`selenium/standalone-chrome`), а результати об'єднуються у зручний HTML-звіт за допомогою **Allure**.

---

## 📋 Попередні вимоги

Перед початком роботи переконайтеся, що на вашому комп'ютері встановлено:

1. **[Docker Desktop](https://www.docker.com/products/docker-desktop/)** (має бути запущений перед виконанням команд).
2. **[Allure Commandline](https://docs.qameta.io/allure/#_installing_a_commandline)** — для генерації та перегляду звітів на вашому комп'ютері:
   * **Mac:** `brew install allure`
   * **Windows:** `scoop install allure` (або `npm install -g allure-commandline`)
3. **Java** (версії 8+) — необхідна для роботи Allure.

---

## ⚙️ Налаштування середовища

1. **Змінні середовища:** Переконайтеся, що у папці `data/` створено файл `.env` з необхідними доступами (логіни, паролі, базові URL).
2. **Файл `.dockerignore`:** Переконайтеся, що в корені проекту є файл `.dockerignore`, який блокує копіювання локального сміття (`.venv`, `chrome/`, `allure-results/`) в Docker-образ, щоб контейнер залишався легким (~400 МБ).

---

## 🏃‍♂️ Запуск тестів

Для старту всього середовища відкрийте термінал у корені проекту та виконайте:

```bash
docker-compose up --build
```

### ЩО ВІДБУВАЄТЬСЯ ПІД ЧАС ЗАПУСКУ:

1. **Піднімається браузер:** Запускається контейнер `chrome-browser` (`selenium/standalone-chrome`).
2. **Перевірка готовності:** Система чекає, поки браузер повністю завантажиться (спрацьовує healthcheck).
3. **Запуск тестів:** Одночасно стартують два ізольованих контейнери: `pytest_runner` та `behave_runner`.
4. **Збереження даних:** Результати автоматично зберігаються на ваш комп'ютер у папки `./allure-results/pytest` та `./allure-results/behave`.

---

## 📊 ФОРМУВАННЯ ТА ПЕРЕГЛЯД ЗВІТІВ (ALLURE)

Коли контейнери з тестами завершать роботу (навіть якщо деякі тести впали), ви можете згенерувати об'єднаний звіт.

Виконайте в терміналі наступну команду:

```bash
allure serve allure-results/pytest allure-results/behave
```

## 🧹 ОЧИЩЕННЯ СЕРЕДОВИЩА (ЗУПИНКА)
Оскільки тестові контейнери є одноразовими процесами, після завершення роботи їх потрібно обов'язково видалити. Щоб звільнити оперативну пам'ять та уникнути помилок конфлікту імен при наступному запуску, виконайте:

```bash
docker-compose down
```
(Підказка: Ви також можете використовувати команду docker-compose up --build --force-recreate, щоб Docker автоматично видаляв старі контейнери перед новим запуском).
